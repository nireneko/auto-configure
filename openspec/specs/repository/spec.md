# Repository Specification

## Purpose
Define the standards for the project repository structure, including file exclusions, binary tracking, and media handling.

## Requirements

### Requirement: Go Build Artifact Exclusion
The repository MUST NOT track binary files or directories generated during the Go build process.

#### Scenario: Ignore bin folder
- GIVEN a directory named `bin/` exists in root
- WHEN running `git status`
- THEN the `bin/` folder MUST NOT be tracked or shown as untracked

#### Scenario: Ignore root binaries
- GIVEN a binary file `so-install` exists in root (built locally)
- WHEN running `git status`
- THEN the `so-install` file MUST NOT be tracked or shown as untracked

### Requirement: Go Test Artifact Exclusion
The repository MUST NOT track output files from the Go test and coverage process.

#### Scenario: Ignore coverage reports
- GIVEN a file `coverage.out` exists
- WHEN running `git status`
- THEN the `coverage.out` file MUST NOT be tracked or shown as untracked

### Requirement: General Tooling Exclusion
The repository SHOULD NOT track files generated by common IDEs (e.g., VS Code, IntelliJ), OS-specific junk (e.g., .DS_Store), or environment files (.env).

#### Scenario: Ignore .DS_Store
- GIVEN a file `.DS_Store` exists
- WHEN running `git status`
- THEN the `.DS_Store` file MUST NOT be tracked or shown as untracked

### Requirement: Media File Exclusion
The repository MUST NOT track screenshots or other non-essential media files that are not part of the application's required assets.

#### Scenario: Identify existing media
- GIVEN the file `error-ddev.png` was previously tracked
- WHEN auditing the repository
- THEN the `error-ddev.png` file MUST BE removed from git tracking

### Requirement: Project Documentation (README)

The repository MUST contain a `README.md` file in the root directory that provides a comprehensive overview of the project.

#### Scenario: Verify README presence
- GIVEN a new contributor clones the repository
- WHEN they look at the root directory
- THEN the `README.md` file MUST be present

#### Scenario: Verify README content sections
- GIVEN the `README.md` file is opened
- WHEN searching for key documentation sections
- THEN it MUST include: Project Description, Installation Modules, System Requirements, and Usage Instructions (Build/Run/Test)

#### Scenario: Verify Installation Modules list
- GIVEN the `README.md` file is opened
- WHEN reviewing the list of what the tool can install
- THEN it MUST include (at a minimum): browsers (Brave, Chrome, Chromium, Firefox), docker, ddev, flatpak, homebrew, nvm, npm, and openvpn
